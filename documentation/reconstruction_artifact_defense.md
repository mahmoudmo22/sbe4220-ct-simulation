- Image size: `256 x 256`

## What the Lines Are

The faint vertical and horizontal lines are reconstruction/simulation artifacts. They are not real structures in the phantom.

They mainly come from:

- discrete ray sampling,
- finite detector bin spacing,
- nearest-neighbor sampling in the forward projector,
- interpolation during backprojection,
- sharp discontinuities in the QA phantom,
- the Ram-Lak filter preserving high-frequency content.

## Short Answer for the Professor

If asked, say:

> The vertical and horizontal lines are artifacts from the simplified discrete simulation and FBP implementation, not physical objects in the phantom. We implemented the projector and reconstructor from scratch on a finite pixel grid, so detector-bin sampling, ray discretization, and interpolation create small structured artifacts. The Ram-Lak filter preserves high frequencies, so these discretization artifacts become more visible.

## Detailed Explanation

### 1. The Continuous CT Model Is Ideal

In theory, a projection is a continuous line integral:

```text
p(theta, s) = integral mu(x, y) dl
```

and FBP reconstructs:

```text
mu(x, y) = integral_0^pi p_filtered(theta, x cos(theta) + y sin(theta)) d theta
```

In the continuous model, if we had infinitely many rays, infinitely many projection angles, and perfect interpolation, the reconstruction would be much smoother.

### 2. The Code Uses a Discrete Pixel Grid

The implementation works on a finite `256 x 256` grid. Rays are sampled at discrete positions, and detector bins are also discrete.

So the continuous line integral becomes an approximation:

```text
line_integral ~= sum(mu(ray_samples)) * pixel_spacing
```

This approximation can introduce preferred grid directions, especially horizontal and vertical directions.

### 3. Forward Projection Uses Simplified Sampling

The forward projector samples the image along each ray. Because this is a from-scratch educational implementation, it uses a simplified sampling strategy rather than a full high-accuracy physical projector.

This can create small discontinuities or step-like changes in the sinogram. When those are backprojected, they can appear as faint lines in the reconstruction.

### 4. Backprojection Uses Interpolation

During backprojection, each pixel maps to a detector coordinate:

```text
s = x cos(theta) + y sin(theta)
```

That detector coordinate often falls between detector bins, so interpolation is needed. Small interpolation errors can accumulate over many angles.

These accumulated interpolation effects may appear as structured lines.

### 5. The QA Phantom Has Sharp Edges

The QA phantom contains:

- a high-contrast rectangular slab,
- a point insert,
- circular low-contrast inserts,
- a sharp outer disk boundary.

Sharp edges contain high-frequency components. These are harder to reconstruct perfectly with finite angular and detector sampling.

The rectangular slab in particular has vertical and horizontal boundaries, so it can produce artifacts aligned with the image grid.

### 6. Ram-Lak Preserves High Frequencies

The Ram-Lak filter is the sharpest filter in the project. It applies a ramp in frequency:

```text
H(w) = |w|
```

This is good for spatial resolution, but it also preserves or amplifies high-frequency components from:

- true edges,
- discretization errors,
- small interpolation errors,
- sampling artifacts.

So with Ram-Lak, artifacts are more visible than they would be with a smoother filter.

If we used Hamming or cosine filtering, the image would look smoother, but spatial resolution would be reduced.

## What Not to Say

Avoid saying:

> The lines are random noise.

That is inaccurate for this figure because no Poisson noise was added.

Avoid saying:

> The lines are real phantom structures.

They are not part of the phantom.

Avoid saying:

> The reconstruction is wrong.

A better explanation is:

> The reconstruction is correct for a simplified discrete simulator, but it contains expected numerical artifacts.

## If Asked How to Reduce the Lines

You can say:

> We can reduce these artifacts by using more projection angles, a smoother reconstruction filter such as Hamming, bilinear interpolation in the forward projector, finer detector sampling, or more advanced reconstruction methods.

Possible improvements:

- increase projections from `180` to `360`,
- use Hamming or cosine filter,
- use bilinear interpolation instead of nearest-neighbor ray sampling,
- increase image resolution,
- use better detector/ray integration,
- apply mild post-reconstruction smoothing for visualization only.

## Why We Still Use This Visual

This visual is still useful because it shows the full FBP pipeline on the QA phantom:

```text
QA phantom -> sinogram -> Ram-Lak FBP reconstruction
```

It also demonstrates an important real lesson:

> Reconstruction quality depends not only on the mathematical formula, but also on sampling, interpolation, and filtering choices.

For a medical imaging course project, this is defensible because the goal is to implement and understand the CT pipeline from scratch, not to match a commercial CT scanner.

## Best One-Minute Defense

Use this if the professor asks directly:

> This reconstruction uses 180 clean parallel-beam projections and Ram-Lak FBP. The vertical and horizontal lines are not dose noise and not phantom structures. They are numerical artifacts caused by discretizing the continuous Radon transform on a finite pixel grid. Our forward projector samples rays through pixels, and backprojection interpolates detector values back into the image. The QA phantom also has very sharp edges, especially the rectangular slab, and Ram-Lak preserves high-frequency content, so those sampling and interpolation artifacts become visible. A smoother filter or higher sampling would reduce them, but Ram-Lak is useful here because it shows the highest-resolution FBP behavior.
